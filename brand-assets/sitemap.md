# ASSUME / ADAPT Website Sitemap (Revised)

Based on UX feedback and practical project constraints, this revised hierarchy reduces overlap and groups content by user intent rather than internal structure.

## 1. Home (`/`)
- **Purpose:** The high-impact landing page.
- **Sections:**
  - Hero Section (Toolbox value proposition).
  - Quick-start CTA (Link to GitHub / Documentation).
  - **News / Updates Feed:** A lightweight section at the bottom showing recent releases, accepted papers, or project milestones.

## 2. The Toolbox (`/toolbox`)
- **Purpose:** The software ecosystem and technical core. Where engineers and users go to understand *how* it works.
- **Sections:**
  - **Core Framework (`assume`):** Details the simulation engine. *This section includes the "Theory & Methodology"* behind the models.
  - **Interface (`assume-gui`):** Highlights the low-code front-end.
  - **Documentation & Resources:** Links to `assume-examples` and external Git repos for explicit **Contributing** guidelines.

## 3. Research & Applications (`/research`)
- **Purpose:** The academic and industry hub. Merges "what we've done" with "who paid for it."
- **Sections:**
  - **Publications & Use Cases:** A unified library. Since academic papers often serve as the primary "use cases" for the tool, this section lists papers with summaries of the *problem they solved*, along with BibTeX links.
  - **Funding Projects:** Dedicated pages for the grants:
    - ASSUME (2021–2025)
    - ADAPT (2025–2028)

## 4. Roadmap (`/roadmap`)
- **Purpose:** The forward-looking timeline.
- **Sections:** 
  - Integrated timeline showing upcoming software features alongside ADAPT research deliverables.

## 5. Community (`/community`)
- **Purpose:** The human element and connection point.
- **Sections:**
  - **People / Team:** PIs, researchers, and partner logos.
  - **Contact:** Form or email links for collaboration.
